前置知识 #

单页应用 #

只有一个html文件 主流的开发模式变成了通过路由进行页面切换

优势: 避免整体页面刷新 用户体验变好

缺点：前端负责事情变多了 开发的难度变大

路由的本质 #

概念来源于后端 : 一个路径表示匹配一个服务器资源 /a.html -> a对应的文件资源 /b.html -> b对应的文件资源

共同的思想: 一对一的关系

前端的路由: 一个路径path对应唯一的一个组件comonent 当我们访问一个path 自动把path对应的组件进行渲染

const routes = [
  {
    path:'/home',
    component: Home
  },
   {
    path:'/about',
    component: About
  },
   {
    path:'/article',
    component: Article
  }
]
